Tournament Commands

ServoBot provides commands to obtain information about certain Magic tournaments. The commands do the following:

  • A link to the standings
  • A link to the pairings
  • A link to decklists
  • Tournament records
  • The record for a particular player
  • A link to the decklist for a particular player

MTGBot Plugin

ServoBot also has an API that can be accessed by other bots, most notably MTGBot. Here's a guide to setting up the commands.

Note: Currently these are experimental. If you notice a problem, please report it in the Discord.

  • Standings
    !addcom !standings %remoteapi https://servobot.info/api/public/melee?query=standings%
    • Links to the full standings for the tournament.
    • Note: !standings is a built in command of MTGBot, and must be overridden by the broadcaster:
      • !override !standings disable
  • Pairings
    !addcom !pairings %remoteapi https://servobot.info/api/public/melee?query=pairings%
    • Links to the pairings for the current round of the tournament.
  • Decklists
    !addcom !decklists %remoteapi https://servobot.info/api/public/melee?query=decks%
    • Links to the list of all decklists.
    !addcom !deck %remoteapi https://servobot.info/api/public/melee?query=deck&name=%input%%
    • Links to the list of a particular player's decklist.
    • If no name is provided, it links to a list of all decklists.
    • The name is passed as an input to the command, for example
      • !deck Brad Nelson
    • The name can be either the player's name or Arena handle.
  • Tournament Records
    !addcom !records %remoteapi https://servobot.info/api/public/melee?query=records%
    • Gives a list of the top few records and how many players are at that record. Useful for estimating what record will be necessary to make a cut to top 8.
    !addcom !record %remoteapi https://servobot.info/api/public/melee?query=record&name=%input%%
    • Gives the record for the player.
    • The name can be either the player's name or Arena handle.
